Personal Points While building this application using the 2 frameworks

Next JS

  • All components are now server-side components, meaning the load speed is faster
  • Newer loading.js file functionality to show an instant loading state from the server. No need to set this up in state management.
  • Fetching of data is easily managed and easy to share across components.
  • With the new update, I don't have to use getServerSideProps anymore since all components are server components by default.
  • I can load fonts from Google fonts with "New @next/font" which is self-hosted. Big win ğŸ’ª
  • We also have async Server Components and enables component-level fetching.
  • New App directory which helps with structuring of code. â¬‡ï¸
  • I have to specifically declare that a component is a client component by add 'use client' as the first line in the component file.
  • I had to adjust quickly to the new App structure that was recently introduced in Next JS 13.

VUE JS

  • Files are nealty organized and easy to find.
  • State syntax made it a bit easier to write.
  • The learning curve should be easier since it has a mix of Ractive JS and follows a similar style.
  • I like that the framework is very flexible and does not follow the opinionated practice of Next JS.
  • Two way binding confused me at times compared to Next JS "one way binding".
  • I got confused sometimes with the components I needed to add into the Parent file.
  • I still have to still explicilty add life cycle hooks.
  • I need to setup routes in the router file and attach various options as well.
